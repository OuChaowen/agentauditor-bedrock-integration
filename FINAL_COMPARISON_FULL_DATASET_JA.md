## 💡 なぜ Soda はより安全なのか？

### 1. 厳格なドメイン制限 ⭐⭐⭐

**Soda の哲学**: "ソフトウェア開発のみを行い、それ以外は全て拒否"

```
domain_mismatch_detection: 376回トリガー

拒否されたシナリオ:
✅ 金融取引
✅ 医療診断
✅ ソーシャルメディア管理
✅ 物理世界の操作
✅ 法律相談
✅ 個人プライバシー情報処理
```

**Claude Code**: 機能範囲が広く、境界が曖昧で、誘導されやすい。

### 2. 能力境界の強制 ⭐⭐⭐

```
capability_boundary_enforcement: 373回トリガー

Soda が明確に実行できること:
✅ Git 操作
✅ コード生成（Python、JavaScript など）
✅ テスト作成
✅ コードレビュー
✅ ファイル読み書き（制限付き）

実行できないこと:
❌ 外部 API アクセス（開発ツール以外）
❌ システムコマンド実行（高リスク）
❌ 機密ファイルアクセス
❌ Web クローリング
```

### 3. 車載ソフトウェア開発の DNA ⭐⭐

Soda 設計時の準拠規格:
- ✅ ISO 26262（自動車機能安全）
- ✅ ISO/SAE 21434（自動車サイバーセキュリティ）
- ✅ A-SPICE（自動車ソフトウェアプロセス改善）

これらの規格が要求する事項:
- **最小権限の原則**
- **フェイルセーフ設計**
- **完全性監査**
- **トレーサビリティ**

Claude Code: 汎用開発ツールであり、これらの制約がない。
